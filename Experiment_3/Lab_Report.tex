%==================================================
\section{Objective}
To implement linear and regularized regression models for predicting a continuous target variable, evaluate their performance using multiple metrics, visualize model behavior, and analyze overfitting, underfitting, and bias--variance characteristics.


%==================================================
\section{Dataset Description}
A real-world regression dataset containing numerical and categorical features related to loan applications is used.  
The target variable is the \textbf{loan amount sanctioned}.


%==================================================
\section{Preprocessing Steps}
Based on the implementation, the following preprocessing steps were carried out:
\begin{enumerate}
    \item \textbf{Data Cleaning}: Duplicate rows were identified using \texttt{df.duplicated()} and removed with \texttt{df.drop\_duplicates()}.
    
    \item \textbf{Missing Values}: Handled by checking for null entries using \texttt{df.isnull().sum()} and removed missing values using SimpleImputer where used median for numerical features and mode for categorical features
    
    \item \textbf{Feature Selection}: Dropped the unnecessary features from the dataframe using \texttt{df.drop(columns = col, inplace = 'True')}
    
    \item \textbf{Handling outliers}: Handled the outliers using capping. \texttt{df.isnull().sum()}

    \item \textbf{Encoding Categorical Variables}: Encoded categorical features using LabelEncoder.
    
    \item \textbf{Feature Scaling}: \item \textbf{Feature Scaling}:  
    Since Linear Regression and its regularized variants (Ridge, Lasso, and Elastic Net) are sensitive to feature magnitudes, \texttt{StandardScaler} was applied to normalize the input features. This ensures that all variables contribute equally to the model, prevents dominance of large-scale features, and enables effective regularization during model training.

    
    \item \textbf{Data Splitting}: The dataset was split into training and testing sets using \texttt{train\_test\_split}.
\end{enumerate}

\subsection*{2. Perform data preprocessing (handling missing values and scaling)}
\begin{lstlisting}[language=Python]
# Checking if any duplicate rows are found.
print("Duplicates in train data: ", any(df_train.duplicated()))
print("Duplicates in test data: ", any(df_test.duplicated()))
# ---- > No duplicates

## Remove unnecessary columns such as Customer ID, Name
cols_to_drop = ['Customer ID', 'Name']
df_train.drop(columns=cols_to_drop, inplace=True)
df_test.drop(columns=cols_to_drop, inplace=True)

# Remove rows with negative Loan Amount
df_train = df_train[df_train["Loan Sanction Amount (USD)"] > 0]

# Checking the missing and null values...for train data
df_train.isnull().sum()

# Checking the missing and null values...for test data
df_test.isnull().sum()

# Handle Missing Values
# dropping rows which has target feature missing
df_train = df_train.dropna(subset=['Loan Sanction Amount (USD)'])

# Numerical Features → Mean
num_train_cols = df_train.select_dtypes(include=["int64","float64"]).columns
num_imputer = SimpleImputer(strategy="median")
df_train[num_train_cols] = num_imputer.fit_transform(df_train[num_train_cols])
num_test_cols = df_test.select_dtypes(include=["int64","float64"]).columns
num_imputer = SimpleImputer(strategy="median")
df_test[num_test_cols] = num_imputer.fit_transform(df_test[num_test_cols])

# Categorical Features → Mode
cat_cols = df_train.select_dtypes(include=["object"]).columns
cat_imputer = SimpleImputer(strategy="most_frequent")
df_train[cat_cols] = cat_imputer.fit_transform(df_train[cat_cols])
df_test[cat_cols] = cat_imputer.transform(df_test[cat_cols])

# Encoding Categorical Variables
le = LabelEncoder()
for col in df_train.select_dtypes(include='object'):
    all_values = pd.concat([df_train[col], df_test[col]])
    le.fit(all_values)
    df_train[col] = le.transform(df_train[col])
    df_test[col] = le.transform(df_test[col])

# Separate features (X) and target (y)
X = df_train.drop("Loan Sanction Amount (USD)", axis=1)
y = df_train["Loan Sanction Amount (USD)"]
\end{lstlisting}

\subsection*{9. Evaluate all models using regression metrics}
\begin{lstlisting}[language=Python]
# Get Best Tuned Models
best_ridge = ridge_grid.best_estimator_
best_lasso = lasso_grid.best_estimator_
best_elastic = elastic_grid.best_estimator_

# Predictions on Validation Set
y_val_pred_lr = reg.predict(X_val_scaled)

y_val_pred_lasso = best_lasso.predict(X_val_scaled)
y_val_pred_elastic = best_elastic.predict(X_val_scaled)

# Define Evaluation Function
def evaluate_model(y_true, y_pred):
    mae = mean_absolute_error(y_true, y_pred)
    mse = mean_squared_error(y_true, y_pred)
    rmse = np.sqrt(mse)
    r2 = r2_score(y_true, y_pred)
    return mae, mse, rmse, r2

lr_metrics = evaluate_model(y_val, y_val_pred_lr)
ridge_metrics = evaluate_model(y_val, y_val_pred_ridge)
lasso_metrics = evaluate_model(y_val, y_val_pred_lasso)

    "Model": ["Linear Regression", "Ridge Regression", "Lasso Regression", "Elastic Net"],

    "MSE": [lr_metrics[1], ridge_metrics[1], lasso_metrics[1], elastic_metrics[1]],
    "RMSE": [lr_metrics[2], ridge_metrics[2], lasso_metrics[2], elastic_metrics[2]],
    "R2 Score": [lr_metrics[3], ridge_metrics[3], lasso_metrics[3], elastic_metrics[3]],

})
performance_table

best_model

\section{Overfitting and Underfitting Analysis}

\begin{itemize}
    \item \textbf{Difference between training and validation errors:}  
    Linear Regression shows a noticeable gap between training and validation performance, indicating sensitivity to noise and potential overfitting. Regularized models such as Ridge, Lasso, and Elastic Net reduce this gap by constraining model coefficients, leading to more stable validation performance.

    \item \textbf{Effect of regularization strength:}  
    Increasing regularization strength (higher $\alpha$) penalizes large coefficients, reducing model complexity. Ridge and Elastic Net demonstrate smoother predictions with controlled coefficients, while excessive regularization may lead to underfitting. Lasso may eliminate less important features entirely when regularization is strong.

    \item \textbf{Improvement in generalization after tuning:}  
    Hyperparameter tuning using GridSearchCV and RandomizedSearchCV significantly improved generalization performance. Ridge and Elastic Net achieved higher validation $R^2$ scores compared to Linear Regression, indicating better balance between bias and variance.
\end{itemize}

\section{Bias--Variance Analysis}

\begin{itemize}
    \item \textbf{Bias behavior of Linear Regression:}  
    Linear Regression exhibits low bias but high variance, as it attempts to fit all features without any constraint. This makes it sensitive to fluctuations in training data, leading to unstable predictions on unseen data.

    \item \textbf{Variance reduction using Ridge and Elastic Net:}  
    Ridge Regression and Elastic Net reduce variance by shrinking coefficient magnitudes. Elastic Net further balances Ridge and Lasso penalties, providing improved stability while retaining important features, resulting in better overall model robustness.

    \item \textbf{Feature sparsity effect in Lasso:}  
    Lasso Regression introduces sparsity by forcing some coefficients to zero. This performs implicit feature selection, simplifying the model and reducing overfitting, but may slightly increase bias if important features are removed.
\end{itemize}

\section{Observations \& Conclusion}

In this experiment, Linear Regression, Ridge Regression, Lasso Regression, and Elastic Net were implemented and evaluated. Linear Regression showed higher variance and weaker generalization due to the absence of regularization. Ridge Regression improved stability by penalizing large coefficients, while Lasso enabled feature selection through coefficient sparsity. Elastic Net provided the best trade-off by combining Ridge and Lasso regularization, achieving superior generalization performance.

Optimal hyperparameters obtained through cross-validation balanced prediction accuracy and model complexity. Overall, Elastic Net emerged as the most reliable model, effectively minimizing overfitting while maintaining predictive accuracy.
