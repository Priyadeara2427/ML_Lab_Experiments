\section{Aim and Objective}
To implement Na\"ive Bayes and K-Nearest Neighbors (KNN) classifiers for a binary classification problem using the Spambase dataset, evaluate them using multiple performance metrics, and analyze their behavior.

%==================================================
\section{Dataset Description}
The experiment uses the \textbf{Spambase Dataset} containing 57 features related to word and character frequencies in emails. The target variable is \texttt{class}, where 1 denotes spam and 0 denotes ham.

%==================================================
\section{Preprocessing Steps}
Based on the implementation, the following preprocessing steps were carried out:
\begin{enumerate}
    \item \textbf{Data Cleaning}: Duplicate rows were identified using \texttt{df.duplicated()} and removed with \texttt{df.drop\_duplicates()}.
    \item \textbf{Missing Values}: Handled by checking for null entries using \texttt{df.isnull().sum()}.
    \item \textbf{Feature Scaling}: Since KNN is distance-dependent, \texttt{StandardScaler} was used to normalize the features.
    \item \textbf{Data Splitting}: The dataset was split into training and testing sets using \texttt{train\_test\_split}.
\end{enumerate}

\subsection*{6. Train Naive Bayes variants}
\begin{lstlisting}[language=Python]
# Standardize numerical features
scaler = StandardScaler()

X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)

gnb = GaussianNB()
mnb = MultinomialNB()
bnb = BernoulliNB()

train_time = [], pred_time = [], accuracy = [], precision = []
recall = [], f1 = [], specificity = [], false_positive_rate = [], report = []

def calculate_metric(models):
    train_time.clear()
    pred_time.clear()
    accuracy.clear()
    precision.clear()
    recall.clear()
    f1.clear()
    specificity.clear()
    false_positive_rate.clear()
    report.clear()
    for model in models:
        start = time.time()
        if model == mnb:
            model.fit(X_train, y_train)
        else:
            model.fit(X_train_scaled, y_train)
        train_time.append(time.time() - start)

        start = time.time()
        if model == mnb:
            y_pred = model.predict(X_test)
        else:
            y_pred = model.predict(X_test_scaled)
        pred_time.append(time.time() - start)

        accuracy.append(accuracy_score(y_test, y_pred))
        precision.append(precision_score(y_test, y_pred))
        recall.append(recall_score(y_test, y_pred))
        f1.append(f1_score(y_test, y_pred))

        cm = confusion_matrix(y_test, y_pred)
        tn, fp, fn, tp = cm.ravel()

        specificity.append(tn / (tn + fp))
        false_positive_rate.append(fp / (fp + tn))

        report.append(classification_report(y_test,y_pred))

models = [gnb, mnb, bnb]
methods = ["Gaussian NB", "Multinomial NB", "Bernoulli NB"]
calculate_metric(models)

NB_table = pd.DataFrame({
    "Metric":["Accuracy", "Precision", "Recall", "F1 Score", "Specificity", "Training Time (s)"],
    "Gaussian NB": [accuracy[0], precision[0], recall[0], f1[0], specificity[0], train_time[0]],
    "Multinomial NB": [accuracy[1], precision[1], recall[1], f1[1], specificity[1], train_time[1]],
    "Bernoulli NB": [accuracy[2], precision[2], recall[2], f1[2], specificity[2], train_time[2]],
})
NB_table

]

\section{Overfitting and Underfitting Analysis}

\subsection*{7.1 Difference between Training and Validation Accuracy}
\begin{itemize}
    \item Training accuracy measures how well the model performs on the data it was trained on, while validation accuracy evaluates performance on unseen data.
    \item A very high training accuracy with significantly lower validation accuracy indicates \textbf{overfitting}, where the model memorizes training data but fails to generalize.
    \item Similar and reasonably high training and validation accuracies indicate good \textbf{generalization}.
    \item Low accuracy on both training and validation datasets indicates \textbf{underfitting}, where the model is too simple to capture underlying patterns.
\end{itemize}

\subsection*{7.2 Effect of Small and Large Values of $k$}
\begin{itemize}
    \item In K-Nearest Neighbors (KNN), a \textbf{small value of $k$} (e.g., $k=1$) makes the model highly sensitive to noise, leading to high variance and overfitting.
    \item A \textbf{large value of $k$} smooths the decision boundary, which may ignore important local patterns and cause underfitting.
    \item Choosing an optimal $k$ balances bias and variance, resulting in better performance on unseen data.
\end{itemize}

\subsection*{7.3 Role of Hyperparameter Tuning in Generalization}
\begin{itemize}
    \item Hyperparameter tuning helps in selecting the best model configuration that minimizes overfitting and underfitting.
    \item Techniques such as Grid Search and Cross-Validation evaluate multiple parameter combinations on validation data.
    \item Proper tuning improves model robustness, predictive accuracy, and ensures better generalization to new datasets.
\end{itemize}


%==================================================
\section{Bias--Variance Analysis}
\subsection*{8.1 Bias Behavior of Na\"ive Bayes}
\begin{itemize}
    \item Na\"ive Bayes assumes strong conditional independence among features, which simplifies the learning process.
    \item Due to this assumption, the model has \textbf{high bias}, as it may not capture complex relationships between features.
    \item However, the high bias makes Na\"ive Bayes stable and less prone to overfitting, especially on small or noisy datasets.
    \item The model performs well when the independence assumption approximately holds.
\end{itemize}
\subsection*{8.2 Variance Behavior of KNN}
\begin{itemize}
    \item K-Nearest Neighbors (KNN) is a non-parametric and instance-based learning algorithm.
    \item For small values of $k$, KNN has \textbf{high variance} because predictions depend heavily on individual training samples.
    \item Noise or outliers in the training data can significantly affect model predictions.
    \item Increasing the value of $k$ reduces variance by averaging over more neighbors.
\end{itemize}

\subsection*{8.3 Effect of Tuning on Bias--Variance Trade-off}
\begin{itemize}
    \item Hyperparameter tuning helps in finding an optimal balance between bias and variance.
    \item In KNN, increasing $k$ increases bias but reduces variance, while decreasing $k$ lowers bias but increases variance.
    \item Proper tuning using cross-validation improves generalization by minimizing overall prediction error.
\end{itemize}

%==================================================
\section{Observations and Conclusion}
\begin{itemize}
    \item Na\"ive Bayes demonstrated fast training and prediction with low computational cost due to its simple probabilistic assumptions.
    \item KNN achieved competitive accuracy when an optimal value of $k$ was selected through hyperparameter tuning.
    \item Grid Search and cross-validation helped identify parameters that improved generalization and reduced overfitting.
    \item Na\"ive Bayes showed better computational efficiency, while KNN required higher memory and computation during prediction.
    \item Overall, both classifiers generalized well when properly tuned, with KNN offering higher flexibility and Na\"ive Bayes providing stable performance.
\end{itemize}

\section{Overfitting and Underfitting Analysis}
The analysis was conducted by comparing training and testing accuracies. A high $k$ value in KNN tended to underfit, while a very low $k$ (e.g., $k=1$) risked overfitting the training noise. The optimal $k$ found through GridSearch provides the best generalization.

%==================================================
\section{Bias--Variance Analysis}
\begin{itemize}
    \item \textbf{Na\"ive Bayes}: High bias due to the independence assumption but low variance.
    \item \textbf{KNN}: Low bias but high variance, which was reduced by selecting an optimal $k$ and using cross-validation.
\end{itemize}

%==================================================
\section{Observations and Conclusion}
The experiment demonstrated that KNN (with KDTree/BallTree) outperformed the Na\"ive Bayes variants in terms of accuracy on the Spambase dataset. Bernoulli NB was the most effective among the probabilistic models for this specific task.
